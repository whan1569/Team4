    # 차량 사고 전조 예측 모델 설계

## 1. 모델 학습 접근 방식
### 1.1. 다 따로 학습해서 합치는 방식 (모듈화 접근)
- **작동 방식**:
  - 사고 전조 요소(차선 이탈, 접근 물체, 도로 상태 등)를 개별적으로 학습.
  - 각 요소의 모델 결과를 결합하여 최종 사고 예측 모델 입력으로 사용.
- **장점**:
  - 개별 요소 최적화 가능 (특화 모델 설계).
  - 병렬 학습 가능.
  - 디버깅이 용이하며 확장성 높음.
- **단점**:
  - 여러 모델 통합 파이프라인 필요.
  - 개별 출력값 결합을 위한 추가 조정 필요.

### 1.2. 한꺼번에 학습하는 방식 (통합 접근)
- **작동 방식**:
  - 모든 사고 관련 데이터를 통합해 하나의 모델로 학습.
  - 입력: 시야 이미지 + 기타 데이터.
  - 출력: 사고 가능성(또는 위험 요소).
- **장점**:
  - 파이프라인 단순화.
  - 요소 간 상관관계를 자동 학습.
  - 실시간 처리에 유리.
- **단점**:
  - 데이터 준비가 복잡.
  - 모델 크기 증가로 학습 및 추론 속도 느림.
  - 디버깅이 어려움.

### 1.3. 추천 방향: 하이브리드 접근
- **1단계**: 개별 요소를 모듈화하여 독립적으로 학습.
- **2단계**: 개별 요소의 결과를 통합해 최종 모델 학습.
  - 개별 모델 출력값을 통합해 머신러닝 모델 학습.
  - 또는 엔드투엔드 통합 학습 진행.

## 2. 모델 설계 예시
### 2.1. 모듈별 학습
- **차선 이탈 감지**:
  - UNet을 사용하여 차선을 감지하고 차량 위치와 차선 간격 계산.
- **물체 접근 감지**:
  - YOLOv8을 사용하여 주변 물체의 거리와 속도를 측정.
- **도로 상태 판단**:
  - SAM(Segment Anything Model)을 사용하여 도로 손상(포트홀, 얼음 등) 여부 감지.

### 2.2. 통합 학습
- **입력**:
  - 차선 이탈 위험도, 물체 접근 위험도, 도로 상태 위험도 등.
- **출력**:
  - 사고 가능성 점수 (0~1).
- **통합 모델 예시**:
  - 간단한 로지스틱 회귀.
  - 복합 구조(CNN + RNN, LSTM 등).

## 3. 결론 및 권장 방법
1. 초기 단계에서는 모듈화된 학습 방식을 사용하여 각 요소의 성능을 최적화.
2. 확장 단계에서 통합 학습 모델을 설계하며, 멀티태스크 학습(Multi-task Learning) 구조를 고려.
3. 데이터 준비와 라벨링, 후처리 과정에서 추가 도움 필요 시 도구와 방법 제공 가능.
